Elementos de mídias: vídeos, imagens e áudios

O HTML oferece suporte a diversos elementos de mídia que permitem incorporar conteúdo multimídia em páginas web. Vamos explorar os elementos mais comuns para inserir imagens, vídeos e áudios em um documento HTML, além de entender como controlar e personalizar esses elementos.

Imagem

O elemento <img> é usado para incorporar imagens em uma página web. A tag <img> não possui uma tag de fechamento, pois é um elemento vazio, o que significa que não contém conteúdo. 

 
                <img src="caminho_da_imagem.jpg" alt="Descrição da imagem" >

O atributo "src" especifica o caminho da imagem e o “alt”, é um texto alternativo, que é exibido caso a imagem não possa ser carregada ou para acessibilidade.

Video

O elemento <video> é usado para incorporar conteúdo de vídeo em uma página web. É possível adicionar várias fontes de vídeo usando as tags <source>, permitindo que o navegador selecione a melhor opção compatível. 

                <video width="640" height="360" controls>
                    < source src="video.mp4" type="video/mp4">
                    < source src="video.webm" type="video/webm">
                </video>

Assim, o navegador selecionará a primeira fonte suportada. O atributo “controls” exibe os controles de reprodução padrão do navegador, como play, pause e volume.

Audio

Para adicionar áudios em uma página web, usamos a tag <audio>, e assim como o vídeo, podemos adicionar várias fontes de áudio usando as tags <source> 

       
                <audio controls>
                    <src="audio.mp3" type="audio/mpeg">
                    <source src="audio.ogg" type="audio/ogg">
                </audio>

Conceitos

Quando acessamos uma página na web, encontramos diferentes conteúdos para interagirmos (textos, imagens, botões, caixas de texto e etc.). Para existirem, os websites são construídos em uma linguagem de marcação chamada HTML que define os elementos e as estruturas das páginas. Os navegadores que utilizamos (Google Chrome, Mozilla Firefox, Opera, Safari, entre outros) são responsáveis por interpretar o código HTML e renderizar os elementos em tela para o usuário.

Uma linguagem de marcação

Diferente das linguagens de programação e algoritmos que seguem uma lógica (revise o conceito de programação nos Fundamentos de Programação), o HTML apenas “marca” quais elementos farão parte da página. Chamamos cada marcação do código de tags. Imagine que essas tags estão destacando qual e onde o elemento será exibido, como se fossem post-its.
